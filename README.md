# Laboratory Journal API

Полнофункциональное приложение ASP.NET Core для управления лабораторным журналом, экспериментами и результатами исследований.

## 📋 Функциональность

### Основные модули:

1. **Система аутентификации и авторизации**
   - Регистрация пользователей
   - Логирование в систему
   - Управление учётными записями
   - Роли (Admin, Researcher, Moderator)
   - Изменение пароля

2. **Управление экспериментами**
   - Создание новых экспериментов
   - Редактирование и удаление экспериментов
   - Отслеживание статуса (Planning, InProgress, Completed, Suspended, Cancelled)
   - Просмотр информации об экспериментах
   - Поиск и фильтрация по названию, описанию, статусу и датам

3. **记録 в лабораторном журнале**
   - Создание записей с разными типами (Observation, Result, Problem, Note, Procedure, Conclusion)
   - Управление приоритетами записей
   - Добавление тегов для организации
   - Поддержка вложений
   - Архивирование записей
   - Полнотекстовый поиск по записям

4. **Управление результатами экспериментов**
   - Регистрация результатов разных типов данных
   - Единицы измерения
   - Статусы результатов (Pending, Verified, Approved, Rejected, Archived)
   - Статистика по результатам
   - История измерений

5. **Модуль поиска**
   - Глобальный поиск по всем данным
   - Расширенный поиск с фильтрами
   - Поиск по тегам
   - Рекомендации на основе истории
   - Пагинация результатов

## 🚀 Установка и запуск

### Предварительные требования:
- .NET 9.0 SDK
- SQL Server LocalDB (устанавливается с Visual Studio)
- Git (для клонирования репозитория)

### Шаги для установки:

1. **Клонирование репозитория**:
```bash
git clone https://github.com/freakuska/Course_for_Yan.git
cd LaboratoryJournal
```

2. **Восстановление зависимостей**:
```bash
dotnet restore
```

3. **Применение миграций базы данных**:
```bash
dotnet ef database update
```

4. **Запуск приложения**:
```bash
dotnet run
```

Приложение будет доступно по адресу: `https://localhost:5001`
Swagger UI: `https://localhost:5001/swagger`

## 📚 API Endpoints

### Аутентификация (`/api/auth`)
- `POST /register` - Регистрация нового пользователя
- `POST /login` - Вход в систему
- `POST /logout` - Выход из системы
- `GET /me` - Получить информацию о текущем пользователе
- `POST /change-password` - Изменить пароль

### Эксперименты (`/api/experiments`)
- `GET /` - Получить список экспериментов текущего пользователя
- `GET /{id}` - Получить информацию о конкретном эксперименте
- `POST /` - Создать новый эксперимент
- `PUT /{id}` - Обновить эксперимент
- `DELETE /{id}` - Удалить эксперимент

### Записи журнала (`/api/journalentries`)
- `GET /experiment/{experimentId}` - Получить записи для конкретного эксперимента
- `GET /{id}` - Получить конкретную запись
- `POST /` - Создать новую запись
- `PUT /{id}` - Обновить запись
- `DELETE /{id}` - Удалить (архивировать) запись
- `GET /search` - Поиск записей

### Результаты экспериментов (`/api/experimentresults`)
- `GET /experiment/{experimentId}` - Получить результаты конкретного эксперимента
- `GET /{id}` - Получить конкретный результат
- `POST /` - Создать новый результат
- `PUT /{id}` - Обновить результат
- `DELETE /{id}` - Удалить результат
- `GET /experiment/{experimentId}/statistics` - Получить статистику результатов

### Поиск (`/api/search`)
- `GET /global` - Глобальный поиск по всем данным
- `POST /experiments` - Расширенный поиск по экспериментам
- `GET /tags` - Поиск по тегам
- `GET /recommendations` - Получить рекомендации

## 🔐 Безопасность

- Требует аутентификации для всех endpoints (кроме регистрации и входа)
- Пользователи видят только свои данные
- Пароли хранятся с использованием хеширования
- CORS политика для защиты от неавторизованного доступа

## 📋 Примеры использования API

### Регистрация:
```bash
curl -X POST https://localhost:5001/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "researcher@example.com",
    "password": "SecurePass123",
    "confirmPassword": "SecurePass123",
    "fullName": "Иван Петров",
    "position": "Старший научный сотрудник"
  }'
```

### Создание эксперимента:
```bash
curl -X POST https://localhost:5001/api/experiments \
  -H "Content-Type: application/json" \
  -H "Cookie: session-id=..." \
  -d '{
    "title": "Анализ воды",
    "description": "Комплексный анализ образцов воды",
    "objective": "Определить качество воды",
    "methodology": "Титрование и спектрофотометрия",
    "startDate": "2026-04-15"
  }'
```

## 📝 Логирование

Приложение логирует:
- Все действия пользователей (вход, создание, обновление, удаление)
- Ошибки при работе с базой данных
- Ошибки аутентификации
- Уровень логирования настраивается в `appsettings.json`

## 🔄 Миграции базы данных

Миграции хранятся в папке `Migrations/` и применяются автоматически при запуске приложения.

Для создания новой миграции:
```bash
dotnet ef migrations add <MigrationName>
```

Для удаления последней миграции:
```bash
dotnet ef migrations remove
```
